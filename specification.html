1.1 Problem Summary

The Recipe Manager is a desktop application designed to help users organize, create, edit, and manage their personal recipes in a structured and efficient manner. The primary purpose is to provide a secure, user-friendly platform for storing recipes with associated ingredients, preparation instructions, and cooking timers, using a MySQL database for data persistence. This eliminates reliance on scattered notes or external websites, enabling quick access and modifications to culinary collections.

Summary of Functions

  • User Authentication System: Secure sign-in/sign-up with password hashing and session management.
  • Recipe Management: Create, view, edit, and delete recipes with details like name, category, prep time, cook time, and instructions.
  • Ingredient Management: Add, update, or delete ingredients with quantities and units for each recipe.
  • Recipe Browsing and Filtering: View and filter recipes by category, name, or full-text search on instructions.
  • File Export/Import: Export recipes to JSON and import from JSON files.
  • Cooking Timer: Built-in timer for prep and cook times.
  • User Account Management: Settings for changing password or email.

Target User Groups

  • Primary Users - Home Cooks: Individuals cataloging family recipes or experimenting with new ones.
  • Secondary Users - Professional Cooks: Small restaurant owners or chefs needing an offline database for quick reference.
  • Tertiary Users - Hobbyists or Students: Learners benefiting from organized storage and timers.

1.2 Research and Motivation

Similar Projects Research

Several platforms address recipe management and organization:

  • Paprika Recipe Manager: Cross-platform app for saving recipes, meal planning, and grocery lists with cloud sync. Focuses on web integration and mobile access.
  • Recipe Keeper: Collects, organizes, and shares recipes across devices, supporting handwritten recipe scanning.
  • Cook'n Recipe Organizer: Desktop software for recipe entry, shopping lists, and nutrition calculation, with strong import features.
  • RecipeSage: Free, web-based tool for recipe storage and sharing, requiring internet connectivity.
  • BigOven: Offers recipe collection, community sharing, and ingredient-based search with a large user base.
  • Apicbase: Professional-grade, cloud-based software for food businesses, providing recipe access for kitchen staff.

References: "The 9 Best Recipe Organizer Apps to Replace Your Cookbooks" (MakeUseOf, 2023); "Review Of The 6 Best Recipe Software Programs" (The Fun Times Guide, 2025); "The best recipe manager for iPhone, iPad, and Mac" (The Sweet Setup, 2020); "5 apps to finally organize your cooking recipes" (Popular Science, 2023); Official websites: paprikaapp.com, recipekeeperonline.com, recipesage.com, dvo.com, get.apicbase.com.

Project Motivation

The Recipe Manager differentiates itself by addressing limitations in existing solutions:

  • Offline and Local Focus: Unlike cloud-reliant apps (e.g., Paprika, Apicbase), it uses secure, local MySQL storage for privacy and offline access.
  • Structured Database Management: Offers relational data handling beyond simple list-based organizers.
  • Integrated Timer and Desktop Experience: Includes a cooking timer and a robust Java desktop interface for non-mobile users.
  • Portability and Customization: Supports JSON export/import and allows personal extensibility without subscriptions.

1.3 Program Functions

Core Functions

  1. User Authentication and Session Management
    • Secure user registration with email uniqueness.
    • Password encryption and comparison.
    • Session management for logged-in users.
  2. Recipe Management
    • Create recipes with name, category, prep time, cook time, and instructions.
    • View, filter, and search recipes by category, name, or instructions.
    • Update and delete recipes.
  3. Ingredient Management
    • Add ingredients to recipes with name, quantity, and unit.
    • View, update, and delete ingredients per recipe.
    • Full-text search on ingredient names.
  4. File Handling
    • Export recipes and ingredients to JSON.
    • Import recipes from JSON files.
  5. User Profile Management
    • Update email or password.
    • Delete user account with cascading data removal.

Advanced Functions

  1. Database Integration
    • MySQL connectivity with JDBC.
    • CRUD operations for users, recipes, ingredients.
    • Foreign key constraints and indexes for efficiency.
    • Automatic schema handling.
  2. Timer Functionality
    • Start countdown timers for prep or cook times.
    • Stop and manage timers.
  3. Security Features
    • Prepared statements to prevent SQL injection.
    • Password encryption.
    • Input validation.

1.5 Permanent Data Required

Users Table - Central user information storage

            userId (int, primary key, auto-increment): Unique identifier.
            username (varchar(50), unique): Login name.
            password (varchar(255)): Encrypted password.
            email (varchar(100), unique): Contact email.
            createdAt (datetime, default CURRENT_TIMESTAMP): Creation timestamp.
            Indexes: On username and email for fast lookups.

Recipes Table - Recipe details storage

            recipeId (int, primary key, auto-increment): Unique identifier.
            userId (int, foreign key references users.userId, on delete cascade): Links to owner.
            name (varchar(200)): Recipe title.
            category (varchar(50)): Classification (e.g., "Dessert", "Main").
            prepTime (int): Preparation minutes.
            cookTime (int): Cooking minutes.
            instructions (text): Step-by-step guide.
            createdAt (datetime, default CURRENT_TIMESTAMP): Creation timestamp.
            updatedAt (datetime, default CURRENT_TIMESTAMP on update): Last modification timestamp.
            Indexes: On userId, category, createdAt (descending), name; full-text on instructions.

Ingredients Table - Ingredient details storage

            ingredientId (int, primary key, auto-increment): Unique identifier.
            recipeId (int, foreign key references recipes.recipeId, on delete cascade): Links to recipe.
            name (varchar(100)): Ingredient name.
            quantity (decimal(10,2)): Amount.
            unit (varchar(20)): Measurement (e.g., "g", "cups").
            Indexes: On recipeId; full-text on name for searches.

Database Relationships and Integrity

  • User-Recipe Relationship: One user can create multiple recipes (1:N).
  • Recipe-Ingredient Relationship: Each recipe can have multiple ingredients (1:N).
  • Referential Integrity: Foreign key constraints ensure data consistency.
  • Cascading Deletes: User deletion removes recipes; recipe deletion removes ingredients.
